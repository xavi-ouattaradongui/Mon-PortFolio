Générateur de nombres premiers optimisé

2025  •  Web App pédagogique

Application didactique et optimisée pour générer et tester des nombres premiers : architecture propre, validation stricte, cache métier et séparation claire des responsabilités (Zustand / TanStack Query / Zod).

Description

Ce TP propose une application complète pour générer et vérifier des nombres premiers en optimisant les performances et en respectant une architecture professionnelle. Le projet illustre la séparation entre la logique métier (Zustand), l’accès asynchrone aux données (TanStack Query), et la validation (Zod). Il inclut un cache interne pour éviter les recalculs, une API simulée, et une interface simple réalisée avec TailwindCSS.

Technologies utilisées

  • Frontend : React (Vite), TailwindCSS, React Hooks
  • State : Zustand (store métier)
  • Asynchrone : TanStack Query (fetch / cache réseau)
  • Validation : Zod
  • Routing : TanStack Router
  • Outils : GitHub, VS Code, ESLint, Prettier

Fonctionnalités principales

  • Générateur aléatoire de nombres via API simulée (TanStack Query)
  • Vérification si un nombre est premier (fonction métier optimisée)
  • Cache interne / memoization pour éviter les recalculs (useMemo / cache métier)
  • Formulaire utilisateur pour saisir un nombre et le valider avec Zod
  • Affichage des états : chargement, succès, échec et validation
  • Navigation : / (accueil) et /primes (page dédiée)

Fonctionnalités avancées / Bonus

  • Cache partagé dans Zustand pour réutiliser des résultats déjà calculés
  • Visualisation simple des étapes de calcul (ex : tentative, cache hit/miss)
  • Tests unitaires pour la fonction `isPrime` (suggestion)

Défis rencontrés

  • Choisir la bonne granularité du cache (mémoire côté store vs memo local)
  • Séparer clairement la logique métier (Zustand) de l’asynchrone (React Query)
  • Valider correctement les données simulées avec Zod (éviter les faux positifs)

Résultat final

Application fonctionnelle et pédagogique : génération fiable de nombres, vérification performante grâce au cache, et architecture modulable claire. Le rendu permet de démontrer les bonnes pratiques (séparation métier/asynchrone/validation) demandées dans l’énoncé du TP.

import { z } from "zod";

export const numberSchema = z.object({
  number: z.number()
});

  number: z.number().min(1).max(50),
});

export async function fetchNumberAlea() {
  await new Promise((resolve) => setTimeout(resolve, 500));
  const raw = { number: Math.floor(Math.random() * 1000) };

  return numberSchema.parse(raw);
}
